# 项目工作流规则

## Git Workflow

- **每改即提交推送**：每次修改完成后，必须 `git add` 对应修改、`git commit` 并 `git push` 到远程仓库。不允许累积多个未提交的修改。

## Problem Analysis Workflow

每次进行问题分析（bug、性能、行为异常等）时，**必须**产出包含以下四部分的报告，缺一不可：

1. **根因分析** — 明确指出直接原因与根本原因，附代码位置与调用链。
2. **修改方案** — 给出最小可行修改，包含具体代码位置与修改内容。
3. **影响面** — 列出修改涉及的相关函数、信号、调用点。
4. **非期待变化** — 评估修改可能造成的副作用与回退策略。

禁止只描述现象或只给出修复建议而不分析影响。上述四部分必须写入对应 `reports/*.md` 文件，并在 `docs/TASK_LOG.md` 中追加记录。
