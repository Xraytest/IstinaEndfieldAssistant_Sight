# Auditor Role

## 目标

审计当前 GUI 代码，识别与 Arknights/Hypergryph 设计规范不一致的地方，以及可改进的交互细节。

## 输入

- `src/gui/pyqt6/` 下所有页面与主题文件
- `docs/design/` 下已有审计报告与研究资料
- 最新的 `docs/TASK_LOG.md`（避免重复已修复问题）

## 输出

向 `docs/design/audits/` 目录追加审计报告，包含：

1. **不一致清单**：当前实现 vs 设计规范的差异（表格形式：文件路径、行号、当前值、期望值、严重度）
2. **交互缺陷**：悬停态、选中态、焦点态、禁用态的缺失或不一致
3. **性能/适配问题**：高分屏、触屏、键盘导航的不足
4. **改进建议**：按 P0/P1/P2 分级，每条建议附带预估工作量

## 约束

- 只读审计，不修改代码
- 优先关注用户可见的视觉/交互问题，而非架构重构
- 已修复过的问题（`docs/TASK_LOG.md` 中有记录）不再重复报告
- 不修改核心业务逻辑（runtime、maa_end、device 等）

## 质量检查

- [ ] 每个问题都有具体文件路径和行号
- [ ] 严重度分级合理（P0 影响使用，P1 影响体验，P2 锦上添花）
- [ ] 已排除用户明确不需要的功能（仪表盘、图表、性能监控）
