## 新发现

### [GUI58-01 Low] `queue_state.py:42-45` — `saved_task_options` 返回浅拷贝，内层字典共享引用

```python
# queue_state.py:42-45
@property
def saved_task_options(self) -> Dict[str, Dict[str, Any]]:
    with self._lock:
        return {k: dict(v) for k, v in self._saved_task_options.items()}
```

**问题**：`dict(v)` 创建了内层字典的浅拷贝，但内层字典中的值（如果本身是可变对象如 list/dict）仍然是共享引用。调用方通过 `saved_task_options` 获取的字典中，修改内层字典的值会绕过 `save_options()` API 直接污染持久化状态：

```python
opts = queue_state.saved_task_options
opts["daily"]["args"] = ["--new-arg"]  # ← 修改了持久化内存中的引用
```

**影响面**：如果有代码路径通过 `saved_task_options` 读取并修改选项值，持久化状态会被静默污染，导致下次 `load()` 时得到不一致的数据。

**建议**：对需要修改的场景，强制使用 `save_options()` API；或在 `saved_task_options` 中返回深拷贝：

```python
import copy
return {k: copy.deepcopy(v) for k, v in self._saved_task_options.items()}
```

### [GUI58-02 Low] `main_window.py:305` — `singleShot` lambda 无父对象，窗口销毁后崩溃风险

```python
# main_window.py:298-305
def _on_execution_state_changed(self, is_executing: bool) -> None:
    ...
    else:
        ...
        self._set_taskbar_progress(100)
        QTimer.singleShot(1000, lambda: self._set_taskbar_progress(0))
```

**问题**：`QTimer.singleShot(1000, lambda: ...)` 创建的 QTimer 没有父对象（由 QApplication 间接管理）。lambda 闭包通过引用捕获 `self`。如果 MainWindow 在 1 秒内被销毁（非托盘模式下的关闭），timer 触发时 lambda 会尝试调用已删除 C++ 对象的方法，导致 `RuntimeError: wrapped C/C++ object of type MainWindow has been deleted`。

**场景重现**：
1. 任务执行完成 → `_on_execution_state_changed(False)` → 创建 1 秒后触发的 timer
2. 用户在 1 秒内关闭窗口（无托盘模式）→ MainWindow 被销毁
3. Timer 触发 → lambda 访问已删除的 `self` → 崩溃

**影响面**：低概率但实际可触发的崩溃。托盘模式下（窗口隐藏而非销毁）此问题不触发，因为 `self` 仍然存活。

**建议**：使用 `QTimer.singleShot` 的方法引用形式（自动持有 self 引用），或在 timer 回调中检查 `self` 是否有效：

```python
QTimer.singleShot(1000, self._set_taskbar_progress, 0)
```

或使用 `sip.isdeleted()` 检查（PyQt6 特有）。

### [GUI58-03 Info] `main_window.py:86-91` — QShortcut 默认上下文限制快捷键范围

```python
# main_window.py:86-91
def _setup_keyboard_shortcuts(self) -> None:
    pages = self._page_stack.count() if self._page_stack else 0
    for i in range(pages):
        shortcut = QShortcut(f"Ctrl+{i+1}", self)
        shortcut.activated.connect(lambda idx=i: self._on_nav_changed(idx))
```

**问题**：`QShortcut` 默认使用 `Qt.ShortcutContext.WidgetShortcut` 上下文。这意味着快捷键只在父控件（QMainWindow）拥有焦点时生效。当用户在子控件中操作时（如在 QLineEdit 中输入命令、在 QTextEdit 中阅读日志），Ctrl+1-5 快捷键完全失效。

Windows 用户期望 Ctrl+数字键切换页面的行为是全局的（类似浏览器标签切换），而非仅在主窗口焦点时生效。

**影响面**：用户输入文本时无法通过快捷键切换页面，必须先用鼠标点击主窗口空白区域再按快捷键，操作体验下降。

**建议**：设置 `WidgetWithChildrenShortcut` 上下文，使快捷键在子控件中也能响应：

```python
shortcut = QShortcut(f"Ctrl+{i+1}", self)
shortcut.setContext(Qt.ShortcutContext.WidgetWithChildrenShortcut)
shortcut.activated.connect(lambda idx=i: self._on_nav_changed(idx))
```
