## 新增发现（1 项）

### DEBOUNCE-LOSS — `SettingsPage` 防抖写入在窗口关闭时丢失未保存编辑

**等级**: 用户体验 / 低
**位置**: `src/gui/pyqt6/pages/settings_page.py:176-179` + `src/gui/pyqt6/main_window.py:179-193`

**问题代码**:

```python
# settings_page.py:176-179 — 防抖写入
def _on_settings_changed(self) -> None:
    # 防抖：按键/值变化频繁时合并为一次实际写入
    self._save_timer.stop()
    self._save_timer.start(400)        # ← 延迟 400ms 写入

# settings_page.py:49-51 — timer 初始化
self._save_timer = QTimer(self)
self._save_timer.setSingleShot(True)
self._save_timer.timeout.connect(self._save_settings)
```

```python
# main_window.py:179-193 — 关闭窗口
def closeEvent(self, event: QCloseEvent) -> None:
    settings = QSettings("ArkStudio", "IstinaEndfieldAssistant")
    settings.setValue("mainWindow/geometry", self.saveGeometry())
    if self._tray_icon is not None and self._tray_icon.is_available():
        event.ignore()
        self.hide()
        ...
    else:
        maaend_page = getattr(self, "_maaend_page", None)
        if maaend_page is not None:
            try:
                maaend_page._persist_state()
            except Exception as exc:
                self._logger.warning(...)
        super().closeEvent(event)
    # ← 无任何 flush _save_timer 或调用 _save_settings 的逻辑
```

**根因分析**:

`SettingsPage` 使用 `QTimer.singleShot(400)` 防抖合并用户编辑，避免每次按键都写入 JSON。但 `closeEvent` 未对 `SettingsPage` 做任何 flush 操作：

1. 用户在设置页修改字段（如 LLM threads 值）
2. `_on_settings_changed()` 重置并启动 400ms timer
3. 用户在 400ms 内关闭窗口（或系统触发关闭）
4. `closeEvent` 仅持久化 `maaend_page` 状态和窗口 geometry，不触及 settings page
5. timer 被 QObject 销毁，`_save_settings()` 永远不执行
6. 下次启动时 `_load_settings()` 读取旧值，用户编辑丢失

**关键细节**: `SettingsPage` 在 `main_window.py:266` 以 `SettingsPage()` 无父引用方式创建，`closeEvent` 中没有获取该实例的引用，无法调用其 `_save_settings()` 方法。

**与同类代码的对比**:

| 页面 | 关闭时持久化 | 防抖 flush |
|------|------------|-----------|
| `SettingsPage` | 无 | 无 |
| `MaaEndControlPage` | `_persist_state()` 在 `closeEvent` 中调用 | 不需要（即时写入） |
| `DeviceSettingsPage` | 无显式持久化 | 即时写入，无防抖 |

**调用链**:
```
用户修改 threads 值 → _on_settings_changed()
  → _save_timer.start(400)
用户关闭窗口（< 400ms）
  → closeEvent → maaend_page._persist_state()
  → super().closeEvent(event)
  → SettingsPage 销毁，timer 未触发
  → _save_settings() 未执行
  → 用户配置丢失
```

**影响面**:
- **低**：防抖窗口仅 400ms，用户连续快速修改后关闭窗口才会触发。但如果用户在切换标签页后直接关闭窗口（不到 400ms），任何未刷新的修改都会丢失。

**修复建议**:

**方案 A** — `MainWindow.closeEvent` 中显式 flush：

```python
# main_window.py:186-193 closeEvent else 分支
else:
    maaend_page = getattr(self, "_maaend_page", None)
    if maaend_page is not None:
        try:
            maaend_page._persist_state()
        except Exception as exc:
            self._logger.warning(...)
    # ← 新增：flush settings page 防抖写入
    settings_page = self.findChild(SettingsPage)
    if settings_page is not None:
        settings_page._save_timer.stop()
        try:
            settings_page._save_settings()
        except Exception as exc:
            self._logger.warning(LogCategory.GUI, "closeEvent 持久化设置失败", error=str(exc))
    super().closeEvent(event)
```

**方案 B** — `SettingsPage` 重载 `closeEvent`（如果作为独立 widget 嵌入）：

```python
# settings_page.py: 新增 closeEvent
def closeEvent(self, event):
    self._save_timer.stop()
    self._save_settings()
    super().closeEvent(event)
```

方案 A 更可靠，因为 `MainWindow.closeEvent` 是统一的关闭入口，不依赖 `SettingsPage` 是否被正确嵌入。
