## §1 新发现

### [NEW-LOW] `tray_icon.py:63-65` — 托盘菜单"退出"按钮实际不退出，仅最小化

```python
# tray_icon.py:63-65
quit_action = QAction(locale.tr("tray_quit"), "Quit")
quit_action.triggered.connect(QApplication.instance().quit)
menu.addAction(quit_action)
```

配合 `main_window.py:108-122` 的 `closeEvent`：

```python
def closeEvent(self, event: QCloseEvent) -> None:
    settings = QSettings("ArkStudio", "IstinaEndfieldAssistant")
    settings.setValue("mainWindow/geometry", self.saveGeometry())
    if self._tray_icon is not None and self._tray_icon.is_available():
        event.ignore()        # ← 忽略关闭事件
        self.hide()           # ← 仅隐藏窗口
        self._tray_icon.show_message(...)
    else:
        # 托盘不可用时才真正退出
        maaend_page = getattr(self, "_maaend_page", None)
        if maaend_page is not None:
            try:
                maaend_page._persist_state()
            except Exception as exc:
                self._logger.warning(...)
        super().closeEvent(event)
```

**问题**：Qt 事件循环中，`QApplication.quit()` 触发 `closeAllWindows()`，进而调用每个顶层窗口的 `closeEvent`。由于 `closeEvent` 在托盘可用时 `event.ignore()` 并隐藏窗口，**"退出"按钮的实际行为是"最小化到托盘"**，而非退出程序。

**调用链推演**：

```
用户点击托盘菜单 "Quit"
  │
  ├── quit_action.triggered 信号
  │     └── QApplication.instance().quit()
  │           └── QApplication.closeAllWindows()
  │                 └── MainWindow.closeEvent(event)
  │                       ├── _tray_icon.is_available() → True
  │                       ├── event.ignore()        ← 关闭被拒绝
  │                       ├── self.hide()           ← 窗口隐藏
  │                       └── show_message("Minimized to tray...")
  │
  └── 用户看到托盘提示"Minimized to tray"，程序仍在运行
```

**影响面**：
- **用户体验**：用户点击"退出"后程序仍在运行，托盘图标持续存在。用户可能认为程序已退出，但进程仍在后台消耗资源。
- **资源泄漏**：CLI 子进程、ADB 连接、scrcpy 会话等资源未释放。
- **队列状态未持久化**：`closeEvent` 在托盘可用路径中**不调用** `maaend_page._persist_state()`，队列状态（当前任务、预设选择、选项配置）未保存。下次启动时队列为空。
- **任务中断**：如果任务正在执行中，点击"退出"仅隐藏窗口，任务继续在后台运行。用户无法通过 GUI 停止任务（窗口已隐藏）。

**对比**：`closeEvent` 的托盘不可用路径（`else` 分支）会持久化队列状态后退出。托盘可用路径反而丢失了这一保护。

**建议**：

方案 1（最小修改）：托盘菜单的"退出"应直接调用 `QApplication.quit()` 并跳过 `closeEvent` 的托盘检查：

```python
# tray_icon.py
quit_action.triggered.connect(self._quit_application)

def _quit_application(self) -> None:
    # 持久化状态
    main_window = self._main_window
    maaend_page = getattr(main_window, "_maaend_page", None)
    if maaend_page is not None:
        try:
            maaend_page._persist_state()
        except Exception as exc:
            logging.getLogger(__name__).warning("Tray quit persist state failed: %s", exc)
    QApplication.instance().quit()
```

方案 2（更优雅）：修改 `closeEvent` 区分"用户主动关闭"与"系统关闭"：

```python
def closeEvent(self, event: QCloseEvent) -> None:
    settings = QSettings("ArkStudio", "IstinaEndfieldAssistant")
    settings.setValue("mainWindow/geometry", self.saveGeometry())
    
    # 检查是否有任务正在执行
    maaend_page = getattr(self, "_maaend_page", None)
    is_executing = getattr(maaend_page, "_is_executing", False) if maaend_page else False
    
    if self._tray_icon is not None and self._tray_icon.is_available() and not is_executing:
        event.ignore()
        self.hide()
        self._tray_icon.show_message(...)
    else:
        if maaend_page is not None:
            try:
                maaend_page._persist_state()
            except Exception as exc:
                self._logger.warning(...)
        super().closeEvent(event)
```

方案 2 同时修复了任务执行中关闭的问题（托盘不可用或任务执行中时直接退出并持久化）。
