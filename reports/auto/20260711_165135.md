### [SG-01 Info] `scene_geometry.py:272` — `_extract_candidates` 阈值计算在平坦图像（std=0）时失效

```python
# scene_geometry.py:272
threshold = float(max(np.percentile(saliency, 89), saliency.mean() + saliency.std() * 0.8))
```

**问题**：当输入图像为平坦/均匀图像（如纯色背景、加载画面）时，显著性图的 `std()` 为 0。此时阈值退化为：

```python
threshold = max(np.percentile(saliency, 89), saliency.mean())
```

如果显著性图本身也是平坦的（所有值相同），`np.percentile(saliency, 89)` 和 `saliency.mean()` 都等于同一个常数。当该常数 ≤ 0 时，`threshold` 可能为 0 或接近 0。

**后果**：
- `mask = (saliency >= threshold).astype(np.uint8) * 255` 在 threshold=0 时产生全白掩码（所有像素通过）
- `cv2.connectedComponentsWithStats` 在全白掩码上产生大量连通域
- 虽然 `area < 350` 和 `area / total_area > 0.35` 过滤掉极小型和超大型区域，但中等大小的连通域仍可能通过筛选
- 最终 `_extract_candidates` 返回大量假阳性候选，消耗后续处理资源并可能误导导航决策

**对比**：批次 34 J04 报告了 saliency 累积乘法过度抑制问题（同一文件 line 266-270），但聚焦于"边缘实体被遗漏"，未涉及平坦图像下阈值计算的失效场景。

**影响面**：低——平坦图像在游戏场景中较少出现（游戏画面通常有丰富纹理），但加载画面、黑场过渡等场景可能触发。

**建议**：在阈值计算中加入下限保护：

```python
std_component = saliency.std() * 0.8
fallback = saliency.mean() + max(std_component, saliency.mean() * 0.1)
threshold = float(max(np.percentile(saliency, 89), fallback))
threshold = max(threshold, 0.05)  # 最低阈值保护
```

### [SRV-02 Info] `scene_service.py:94` — `verify_by_key_elements` 子串匹配产生假阳性

```python
# scene_service.py:89-95
def verify_by_key_elements(
    self, screen: np.ndarray, expected_templates: List[str]
) -> Tuple[bool, List[ElementInfo]]:
    elems = self.recognizer.recognize_templates(screen, names=expected_templates)
    found_names = {e.label for e in elems}
    all_found = all(any(t in fn for fn in found_names) for t in expected_templates)
    return all_found, elems
```

**问题**：`any(t in fn for fn in found_names)` 使用 Python 子串匹配（`in` 操作符），而非精确匹配。这导致：

1. **前缀/后缀误匹配**：`expected_templates=["button"]` 时，识别到的 `"toolbutton"` 标签会使 `"button" in "toolbutton"` 为 `True`，误判为找到目标。
2. **部分匹配**：`expected_templates=["menu"]` 时，`"menu_button"`、`"submenu_arrow"` 都会通过校验。
3. **语言/变体混淆**：中文标签中，"关闭按钮" 包含 "按钮"，但 "确认按钮" 也包含 "按钮"，当期望的是特定按钮时无法区分。

**影响面**：中——`verify_by_key_elements` 用于页面验证（`verify` 方法的替代路径）。假阳性会导致系统认为已到达目标页面，实际仍在错误页面，触发后续自动化流程在错误上下文中执行。

**建议**：改为精确匹配或前缀匹配：

```python
all_found = all(
    any(fn == t or fn.startswith(t + "_") or fn.endswith("_" + t) for fn in found_names)
    for t in expected_templates
)
```

或更严格地，要求 `fn` 与 `t` 完全一致或以 `t` 为独立词：

```python
import re
all_found = all(
    any(re.fulljoin(re.escape(t), fn) or re.search(r'\b' + re.escape(t) + r'\b', fn)
    for fn in found_names)
    for t in expected_templates
)
```
