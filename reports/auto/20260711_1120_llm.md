### [LLM01 High] `llm/runtime.py:131-133` CUDA 失败后强制 CPU 重试丢失原始错误

**位置**：`src/core/capability/llm/runtime.py:131-133`

```python
ok = self._try_start(exe, model_path, llm_cfg)
if not ok and not self._cuda_failed:
    self._cuda_failed = True
    return self._try_start(exe, model_path, llm_cfg, force_cpu=True)
return ok
```

**根因分析**：当第一次启动（GPU 模式）失败时，代码设置 `_cuda_failed = True` 并立即重试（CPU 模式）。但：
1. **原始错误丢失**：`_try_start` 返回 `False` 但不说明失败原因（OOM？模型加载失败？CUDA 驱动问题？）。用户无法诊断 GPU 模式失败的原因。
2. **CPU 重试可能也失败**：如果失败原因是模型文件损坏或路径错误，CPU 重试也会失败，用户看到两个连续的失败而无任何解释。
3. **`_cuda_failed` 永久为 True**：一旦设置，永远不会重置。如果用户修复了 GPU 问题（如更新驱动），`start()` 仍然会跳过 GPU 模式。

**影响面**：LLM 服务启动。用户遇到 GPU 问题时无法获得诊断信息，且无法在修复后自动重试 GPU 模式。

**修改方案**：
```python
ok = self._try_start(exe, model_path, llm_cfg)
if not ok and not self._cuda_failed:
    self._cuda_failed = True
    self._logger.warning("GPU 模式启动失败，尝试 CPU 模式...")
    return self._try_start(exe, model_path, llm_cfg, force_cpu=True)
return ok
```

And add a method to reset `_cuda_failed` when config changes.

### [LLM04 Info] `llm/runtime.py:204-217` `_find_pids_on_port` 使用 `netstat -ano` 解析

**位置**：`src/core/capability/llm/runtime.py:204-217`

```python
@staticmethod
def _find_pids_on_port(port: int) -> set[int]:
    try:
        output = subprocess.check_output(["netstat", "-ano"], timeout=5, text=True)
        pids: set[int] = set()
        for line in output.splitlines():
            if f":{port}" in line and "LISTENING" in line:
                parts = line.strip().split()
                if parts:
                    try:
                        pids.add(int(parts[-1]))
                    except ValueError:
                        pass
        return pids
    except Exception:
        return set()
```

**根因分析**：`netstat -ano` 输出格式在不同 Windows 版本中可能不同。例如，Windows 10/11 的 `netstat` 输出可能包含 IPv6 地址（如 `[::]:9998`），其中 `:9998` 是地址的一部分而非端口。`f":{port}" in line` 会错误匹配 `[::]:9998`。

**影响面**：低——在单端口（如 9998）场景下，`[::]:9998` 和 `0.0.0.0:9998` 都指向同一服务，误匹配不会导致错误终止。
