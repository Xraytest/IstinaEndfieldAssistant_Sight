**修改建议**：
逐行修正 6 处文件路径引用。修正后的对照表：

### [DOC-03] `ARCHITECTURE.md` 目录结构图与实际目录结构不一致

**优先级：Medium**
**位置**：`docs/ARCHITECTURE.md:26-34`

```markdown
src/
├── core/                  # core business logic
│   ├── capability/        # capability layer (device, recognition, navigation, LLM)
│   ├── service/           # service layer (scene understanding, task scheduling)
│   └── runtime/           # runtime (IstinaRuntime facade)
├── cli/                   # CLI entry point and command dispatch
├── gui/                   # GUI (PyQt6)
└── infra/                 # infrastructure (config, logging, i18n)
```

**实际目录结构**：
```
src/
├── __init__.py
├── cli/                   # CLI entry point and command dispatch
├── core/
│   ├── __init__.py
│   ├── capability/
│   │   ├── device/        # ADB, touch, android_runtime
│   │   ├── element_recognition/  # Template/OCR/Color/YOLO + Pipeline
│   │   └── llm/           # LlamaServerRuntime, LlmClient
│   ├── foundation/        # paths, logger, gpu_check
│   └── service/
│       ├── maa_end/       # MaaEndRuntime
│       ├── navigation/    # Navigator, VlmWalkNavigator, etc.
│       └── runtime.py     # IstinaRuntime facade
├── gui/                   # GUI (PyQt6)
```

**不一致之处**：
1. `infra/` 目录**不存在** — 实际为 `core/foundation/`
2. `runtime/` 子目录**不存在** — `IstinaRuntime` 在 `core/service/runtime.py`
3. `capability/` 下缺少 `element_recognition/` 子目录层级
4. `service/` 下缺少 `maa_end/` 子目录
5. `config/` 和 `assets/`（项目根级别）未被提及

**影响面**：目录结构图是开发者理解项目组织的第一视觉入口，错误结构导致开发者无法通过文档导航到源码。

**修改建议**：更新目录结构图为实际结构，或在图下注明"为简化表示，部分子目录未展开"。

### [DOC-04] `docs/README.md` 记录 Bug 2（预览定时器干扰任务执行）已被代码修复，但文档未更新

**优先级：Low**
**位置**：`docs/README.md:82-108, 252`

README Bug 2 描述：预览定时器（1.5s interval）在任务执行期间持续触发 `_sync_execute("screenshot")`，干扰 CLI 进程命令队列。

**当前代码修复状态**：
`src/gui/pyqt6/main_window.py:327-329`：

```python
if self._maaend_page._is_executing:
    self._logger.debug(LogCategory.GUI, "预览退出: 任务执行中")
    return
```

`_refresh_preview` 在发起 screenshot 前检查 `_is_executing` 标志，若任务正在执行则**直接返回**。这意味着预览定时器虽持续触发，但不产生实际的 screenshot 子进程调用，**Bug 2 已被代码修复**。

然而，README 第 252 行仍写道：
> Bug 2 (preview-timer interference) still requires pausing the preview during task execution

这与实际代码状态不一致。修复方式（通过 `_is_executing` 标志守卫）与 README 建议的 `stop_preview_timer()` / `start_preview_timer()` 不同，但效果等价。

**修改建议**：更新 README Bug 2 状态为"已修复（通过 `_is_executing` 守卫）"，或删除该条目。

### [DOC-05] `docs/README.md` 记录 Bug 3（预设类型硬编码为 "task"）仍有效，但修复建议不完整

**优先级：Low**
**位置**：`docs/README.md:111-137`、`src/gui/pyqt6/pages/maaend_control_page.py:701,1389,1432`

README Bug 3 描述：预设展开为任务条目时 `type` 硬编码为 `"task"`，导致 `_runtime_queue_runner` 中的 `if item_type == "preset"` 分支为死代码。

**当前代码修复状态**：
`_runtime_queue_runner`（`maaend_control_page.py:824`）中**确实没有** `item_type` 分支判断——所有条目统一执行 `self._sync_execute(f"task run {clean_name}", ...)`。`type` 字段仅在 `_format_queue_label`（`maaend_control_page.py:1034-1035`）中用于显示：

```python
def _format_queue_label(self, name: str, item_type: str, options: Dict[str, Any]) -> str:
    return f"[{item_type.upper()}] {_zh(name)}"
```

**影响面**：
- 预设展开的任务在队列中显示为 `[TASK]` 而非 `[PRESET]`，用户无法区分手动添加的任务和预设展开的任务。
- 若未来需要支持"直接在队列中运行预设"（不展开），当前数据结构不支持——`type` 字段被误用。

**修改建议**：README 建议"删除死代码或加注释"，但更完整的修复是：
1. 预设展开时使用 `"type": "preset"` 保持原始类型信息
2. `_runtime_queue_runner` 增加 `if entry.get("type") == "preset":` 分支执行 `preset run`
3. `_format_queue_label` 已能正确显示类型
