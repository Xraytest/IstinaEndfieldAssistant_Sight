## 新增发现（1 项）

### BACKEND-SILENT — `template_backend.py` / `color_backend.py` 非主识别路由 bare except: pass 静默吞错

**等级**: 代码质量 / 低
**位置**: `src/core/capability/element_recognition/backends/template_backend.py:248-249, 283-284`, `src/core/capability/element_recognition/backends/color_backend.py:244-245`

**问题代码**:

```python
# template_backend.py:248-249 — Route 1: Legacy TemplateMatcher
except Exception:
    pass  # ← 静默吞掉，无日志

# template_backend.py:283-284 — Route 2: RecognitionEngine SIFT
except Exception:
    pass  # ← 静默吞掉，无日志

# color_backend.py:244-245 — Route 1: RecognitionEngine
except Exception:
    pass  # ← 静默吞掉，无日志
```

**根因分析**:

`TemplateBackend._match_single` 和 `ColorBackend._match_one_signature` 各有多条识别路由（Pipeline → Legacy → SIFT / RecognitionEngine → OpenCV）。非主路由在异常时使用 bare `except: pass` 静默吞掉：

| 文件 | 路由 | 异常处理 | 主路由 |
|------|------|---------|-------|
| `template_backend.py:248` | Legacy TemplateMatcher | `except Exception: pass` | Pipeline (line 204) |
| `template_backend.py:283` | RecognitionEngine SIFT | `except Exception: pass` | Pipeline (line 204) |
| `color_backend.py:244` | RecognitionEngine | `except Exception: pass` | OpenCV (line 248) |

当非主路由异常时（如模板文件损坏、SIFT 引擎内存不足、RecognitionEngine 连接中断），调用方 `_match_single` 返回 `None`，外观上与"模板不存在"完全一致。用户/开发者无法区分：

1. 模板确实不存在 → 正常行为
2. 模板存在但匹配引擎崩溃 → 静默失败，无诊断

**调用链**:
```
识别请求 → TemplateBackend.recognize()
  → _match_single(screen, template_name)
    → Route 0 (Pipeline): 正常匹配成功 → 返回 ElementInfo
    → Route 1 (Legacy): 异常 → pass → 返回 None（外观同"未匹配"）
    → Route 2 (SIFT): 异常 → pass → 返回 None（外观同"未匹配"）
  → 调用方收到 None，以为模板不存在
```

**影响面**:
- **低**：主路由（Pipeline + OpenCV）正常工作，非主路由仅在降级时使用。但调试时无法定位非主路由失败原因，增加排障成本。

**修复建议**:

```python
# template_backend.py:248-249
except Exception as e:
    logger.debug("Legacy matcher failed for %s: %s", template_name, e)

# template_backend.py:283-284
except Exception as e:
    logger.debug("SIFT recognition failed for %s: %s", template_name, e)

# color_backend.py:244-245
except Exception as e:
    logger.debug("RecognitionEngine color match failed: %s", e)
```

最小修改：将 `except Exception: pass` 改为 `except Exception as e: logger.debug(...)`，保留降级行为但提供诊断信息。
