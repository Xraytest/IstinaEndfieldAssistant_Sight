## 新增发现（2 项）

### PRTS-WORKER-LINGER（QThread 生命周期未管理）— `prts_full_intelligence_page.py`

**等级**: 代码错误 / 中
**位置**: `src/gui/pyqt6/pages/prts_full_intelligence_page.py:246-249, 1726`

**问题代码**:

```python
# prts_full_intelligence_page.py:246-249
self._worker = LlmChatWorker(self._bridge, text, image_b64, self)
# worker 仅用于在独立线程触发 execute（避免阻塞 UI）；
# 实际结果由 bridge.commandFinished 信号回传，_on_command_finished 处理。
self._worker.start()

# prts_full_intelligence_page.py:1726 — 无 closeEvent/hideEvent/destroyed 处理
```

**根因分析**:

`LlmChatWorker` (QThread) 在 `_send_chat()` 中启动（line 249），但整个页面类**无任何生命周期清理方法**：

1. 页面继承 `QWidget`，未实现 `closeEvent`、`hideEvent` 或连接 `destroyed` 信号
2. `_startup_timer` (QTimer, line 179) 在 `_start_llm()` 中启动后无清理路径
3. worker 线程仅在 `_on_command_finished` 信号回传后将 `self._worker` 置 None（line 224），但**不等待线程退出**
4. 如果用户切换页面标签、关闭窗口或在 worker 运行期间页面被销毁，worker 线程仍活跃

**与正确实现的对比**:

| 对比点 | `maaend_control_page.py`（正确） | `prts_full_intelligence_page.py`（缺失） |
|--------|-------------------------------|----------------------------------------|
| closeEvent | ✅ line 1728 | ❌ 无 |
| worker 停止 | ✅ `_worker.stop()` → `_worker.wait()` → `_worker.deleteLater()` | ❌ 仅 `self._worker = None` |
| 启动前状态检查 | ✅ `_worker and getattr(self._worker, '_stopped', False)` | ❌ 仅 `self._worker is not None` |
| timer 清理 | ✅ closeEvent 中清理 | ❌ 无 |

**调用链（崩溃场景）**:

```
场景：用户发送聊天请求后立即切换标签页
  1. _send_chat() → LlmChatWorker.start() → worker 线程开始执行
  2. 用户切换到其他页面 → PrtsFullIntelligencePage 被隐藏/销毁
  3. worker 线程执行 self._bridge.execute("llm chat", ...)
  4. bridge 执行完成 → commandFinished 信号发射
  5. _on_command_finished 被调用（PyQt 自动排队到主线程）
  6. line 222: self._send_btn.setEnabled(True) — 但按钮已被销毁！
  7. → RuntimeError: wrapped C/C++ object of type QPushButton has been deleted
```

**影响面**:
- **中**：用户切换页面时可能触发 `RuntimeError: wrapped C/C++ object has been deleted`。概率取决于 LLM 响应延迟——短响应下 worker 在页面隐藏前完成，长响应下崩溃概率上升。
- 低概率不等于低严重：Qt 的 `RuntimeError` 会终止 PyQt 事件循环，导致整个 GUI 冻结。

**修复建议**:

```python
# 方案 1（最小修改）：添加 closeEvent 清理
def closeEvent(self, event):
    super().closeEvent(event)
    if self._startup_timer is not None:
        self._startup_timer.stop()
    if self._worker is not None:
        self._worker.wait(3000)  # 等待最多 3 秒

# 方案 2（更安全）：使用 worker.finished 信号连接 deleteLater
# 在 _send_chat 中：
self._worker = LlmChatWorker(self._bridge, text, image_b64, self)
self._worker.finished.connect(self._worker.deleteLater)
self._worker.start()

# 并在 closeEvent 中：
def closeEvent(self, event):
    super().closeEvent(event)
    if self._startup_timer is not None:
        self._startup_timer.stop()
    # 等待 worker 自然完成，不强制 terminate
    if self._worker is not None:
        self._worker.wait(5000)
```

**推荐方案 1**（最小可行修改，与 maaend_control_page 模式一致）。
