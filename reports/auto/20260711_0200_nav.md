## 1. 代码错误（功能失效类）

### [NAV-01] `find_by_name` 空字符串匹配所有实体，语义违背且可能引发意外批量操作
**优先级：Medium**
**位置**：`src/core/service/navigation/entity_db.py:129-133`

```python
pattern = re.compile(re.escape(name), re.IGNORECASE)
results = []
for key, entities in self._by_name.items():
    if pattern.search(key):
        results.extend(entities)
```

`re.escape("")` 返回空串 `""`，`pattern.search(key)` 对任意字符串在位置 0 即匹配成功。因此 `find_by_name("")` 返回 `_by_name` 中**全部实体**（截断至 `limit`）。调用方（如 `to_entity` 的 name_filter）若传入空字符串或经过 `.strip()` 后为空，会触发全量遍历，浪费 CPU 并可能对全部实体执行导航。

**修改建议**：在入口校验 `name` 非空：
```python
if not name or not name.strip():
    return []
```
或对空串返回精确匹配结果（仅 `key_name == ""` 的实体，若有）。

### [NAV-02] `to_coords` 对 `map_id="unknown"` 强制传送，与 `to_coords_vlm` 行为不一致
**优先级：Medium**
**位置**：`src/core/service/navigation/navigator.py:82`

```python
if current_pos and current_pos.map_id != map_name:
```

### [NAV-03] `Navigator.__init__` 忽略 `EntityDatabase.load()` 返回值，静默接受空数据库
**优先级：Medium**
**位置**：`src/core/service/navigation/navigator.py:41`

```python
self._entities = EntityDatabase()
self._locator = MinimapLocator(self._data)
self._entities.load()
```

`EntityDatabase.load()` 在 `maaend_entities.json` 缺失或格式错误时返回 `False`，但 `__init__` 不检查返回值。后续 `to_entity`、`where_am_i`、`list_entities` 等调用均依赖该数据库；若加载失败，这些方法返回空结果或 "entity not found"，调用方无法区分"确实没有实体"与"数据库加载失败"。

**修改建议**：检查返回值并在失败时记录错误或抛出异常：

### [NAV-04] `EntityDatabase.load()` 对 JSON 结构无 schema 校验，畸形文件导致 AttributeError
**优先级：Medium**
**位置**：`src/core/service/navigation/entity_db.py:85-99`

`json.load` 的 `try/except` 仅包裹文件读取与解析（line 78-83），后续的 `for map_entry in raw_maps` 循环未受保护。若 `maaend_entities.json` 顶层为 `{}`（dict）而非 `[]`（list），`map_entry` 变为字符串键名，`map_entry.get("map_id", "")` 触发 `AttributeError`。同理，若 `levels` 字段缺失或为 `null`，内层循环也会崩溃。

**修改建议**：在循环入口校验类型，或扩大 try/except 覆盖数据处理逻辑：
```python
if not isinstance(raw_maps, list):
    self._logger.error("invalid entity db format: expected list")
    return False
```

## 2. 漏洞 / 健壮性问题

### [NAV-05] `Entity.from_raw` 对原始值类型无校验，畸形数据导致 TypeError
**优先级：Low**
**位置**：`src/core/service/navigation/entity_db.py:27-40`

```python
rl = raw.get("raw_location", [0, 0, 0])
...
raw_location=(float(rl[0]), float(rl[1]), float(rl[2])) if len(rl) >= 3 else (0.0, 0.0, 0.0)
```

`.get()` 的默认值 `[0, 0, 0]` 仅保护缺失键的情况。若 JSON 中 `raw_location` 为字符串 `"1,2,3"` 或 dict `{"x":1,"y":2}`，`len(rl)` 对 dict 返回键数、对字符串返回字符数，随后 `rl[0]` 对 dict 返回键名（字符串）、对字符串返回首字符——`float(rl[0])` 全部失败并抛出 `TypeError`。异常传播至 `load()` 循环，导致整个数据库加载中断。

**修改建议**：在 `from_raw` 中校验类型并跳过畸形条目，或记录警告后使用默认值。

### [NAV-06] `EntityDatabase.load()` 并发调用时重复加载，索引被覆盖
**优先级：Low**
**位置**：`src/core/service/navigation/entity_db.py:69-107`

`_loaded` 标志的检查与设置非原子操作。若两线程同时首次调用 `load()`，两者均看到 `_loaded == False`，并发执行文件读取与索引构建。后完成的线程覆盖先完成的线程的 `_by_id`、`_by_name` 等字典，但 `_entities` 列表是 `append`，导致最终列表包含重复实体。结果：`count()` 偏大、`_by_id` 取到最后写入的实例、`find_by_name` 返回去重前的全量列表。

**修改建议**：用 `threading.Lock` 保护 `load()`，或在 `_ensure_loaded` 中实施 DCL。

### [NAV-07] `MapDataLoader.load_layout` 对 JSON 数组元素无类型校验
**优先级：Low**
**位置**：`src/core/service/navigation/map_data_loader.py:105-107`

```python
for lid, lv in raw.get("levels", {}).items():
    levels[lid] = MapLevel(x=lv["x"], y=lv["y"], width=lv["width"], height=lv["height"])
```

若 `lv["x"]` 为字符串 `"100"` 而非整数，`MapLevel` 接受该值（dataclass 不强制类型），后续坐标运算产生隐式类型错误。若 `lv` 缺少 `"x"` 键则 `KeyError` 被外层 `except` 捕获，但整次加载失败、该地图的 layout 为空。

**修改建议**：在循环内校验并钳制数值类型，或对单条畸形数据记录警告后跳过。

### [NAV-08] `MapDataLoader.load_grid_tiers` 返回浅拷贝，内部 `GridCell` 对象可被外部修改
**优先级：Low**
**位置**：`src/core/service/navigation/map_data_loader.py:134 / 149-154`

```python
def load_grid_tiers(self) -> Dict[str, GridCell]:
    ...
```

返回的字典是新对象，但其 `GridCell` 值（`center`、`lb`、`rt`、`items`）均为引用。调用方修改 `grid_cell.items["new"] = "x"` 会直接修改缓存中的 `_grid_tiers`，污染后续查询结果。

### [NAV-09] `MapDataLoader.load_all_layouts` 静默忽略单文件失败，调用方无法获知哪些地图可用
**优先级：Low**
**位置**：`src/core/service/navigation/map_data_loader.py:120-123`

```python
def load_all_layouts(self) -> Dict[str, MapLayout]:
    for name in _LAYOUT_FILES:
        self.load_layout(name)
    return dict(self._layouts)
```

`load_layout` 在文件缺失或 JSON 错误时返回 `None` 并记录 warning，但 `load_all_layouts` 不收集这些失败。调用方拿到 `_layouts` 后无法知道哪些地图实际加载成功。若仅 `map02` 损坏，调用方看到 `_layouts` 只有 2 项，但不知晓具体缺失。

**修改建议**：返回加载结果列表或 raise 聚合异常。

### [NAV-10] 所有 `load_*` 方法使用裸 `except Exception`，吞掉 `MemoryError` / `KeyboardInterrupt`
**优先级：Low**
**位置**：`src/core/service/navigation/map_data_loader.py` 多处（101, 144, 179, 208）

```python
except Exception as exc:
    self._logger.warning("failed to load ...: %s", exc)
    return ...
```

`except Exception` 捕获 `MemoryError`、`RecursionError` 等系统级异常（注：`KeyboardInterrupt` 和 `SystemExit` 继承 `BaseException` 而非 `Exception`，因此不会被捕获。但 `MemoryError` 会被捕获）。在内存不足时，本应让进程快速失败而非静默回退空数据。

**修改建议**：缩小异常捕获范围至 `(OSError, json.JSONDecodeError, ValueError)`。

## 3. 可提升体验的细节（非错误）

### [NAV-11] `list_entities` 每次调用无条件触发 `load()`，无意义冗余
**位置**：`src/core/service/navigation/navigator.py:164`

```python
def list_entities(...):
    self._entities.load()    # 已加载时立即返回 True，但仍有一次函数调用开销
```

`EntityDatabase.load()` 在已加载时仅做一次布尔判断并返回，开销极低，不构成性能问题。但语义上暗示"每次查询前需重新加载"，易误导维护者以为数据库支持热更新。当前无热更新机制，`load()` 仅在首次生效。

**修改建议**：删除冗余调用，或改用 `_ensure_loaded()` 明确表达惰性加载意图。
