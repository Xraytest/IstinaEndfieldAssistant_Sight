### [GEO01 Medium] `scene_geometry.py:134-136` Canny 边缘检测对 float32 输入截断为 uint8

**位置**：`src/core/capability/element_recognition/backends/scene_geometry.py:134-136`

```python
def _compute_edge_map(self, gray: np.ndarray) -> np.ndarray:
    blurred = cv2.GaussianBlur(gray, (0, 0), 2.0)
    edge = cv2.Canny(blurred.astype(np.uint8), 60, 160).astype(np.float32) / 255.0
    return cv2.GaussianBlur(edge, (0, 0), 3.0)
```

**根因**：
`gray` 参数为 float32（line 50: `.astype(np.float32)`），值域 [0.0, 255.0]。`GaussianBlur` 输出也是 float32。`astype(np.uint8)` 做**截断**（非四舍五入），值 < 0 回绕为 255，值 > 255 回绕为 0。对 Canny 的 60/160 双阈值来说，边缘像素的微小精度损失通常不改变检测结果，但：
1. 图像边界处 BORDER_REFLECT_101 可能产生负值 → 回绕为 255 → Canny 误检为边缘
2. 代码语义错误——对 float 图像直接 cast 而非 clamp + round

**影响面**：`_compute_edge_map` 的输出用于 `_extract_candidates` 的边缘评分（line 308 `edge_map.mean()`）和 `_estimate_ground` 的纹理差值（line 200）。边缘图精度下降会轻微影响候选区域评分和地面线检测。

**修改方案**：
```python
def _compute_edge_map(self, gray: np.ndarray) -> np.ndarray:
    blurred = cv2.GaussianBlur(gray, (0, 0), 2.0)
    blurred_clipped = np.clip(blurred, 0, 255).astype(np.uint8)
    edge = cv2.Canny(blurred_clipped, 60, 160).astype(np.float32) / 255.0
    return cv2.GaussianBlur(edge, (0, 0), 3.0)
```

### [GEO02 High] `scene_geometry.py:50-52` BGRA 输入导致 `cvtColor(COLOR_BGR2GRAY)` 崩溃

**位置**：`src/core/capability/element_recognition/backends/scene_geometry.py:50-52`

```python
h_img, w_img = screen.shape[:2]
gray = cv2.cvtColor(screen, cv2.COLOR_BGR2GRAY).astype(np.float32)
hsv = cv2.cvtColor(screen, cv2.COLOR_BGR2HSV).astype(np.float32)
```

**根因**：与 `matcher.py`（batch 45 SCR02）完全相同的模式。若 `screen` 为 BGRA（4 通道，如 ADB screencap 在某些设备/驱动下返回），`COLOR_BGR2GRAY` 只接受 3 通道 BGR 输入，`cv2.cvtColor` 抛 `cv2.error: Assertion failed (cn == 3)`。

**影响面**：`SceneGeometryAnalyzer.analyze()` 通过 `SceneUnderstandingService.analyze_scene_3d()` 暴露。当前该方法是**死代码**（从未被调用，见 [DEADCODE01]），因此当前不触发。但一旦未来启用 3D 场景分析，BGRA 截图将直接崩溃。

**修改方案**：
```python
if screen.shape[-1] == 4:
    screen = cv2.cvtColor(screen, cv2.COLOR_BGRA2BGR)
gray = cv2.cvtColor(screen, cv2.COLOR_BGR2GRAY).astype(np.float32)
hsv = cv2.cvtColor(screen, cv2.COLOR_BGR2HSV).astype(np.float32)
```

### [DEVICE01 High] `device_settings_page.py:209` `QPropertyAnimation` 在 `QLabel` 上使用不存在的 `windowOpacity` 属性

**位置**：`src/gui/pyqt6/pages/device_settings_page.py:92,209-215`

```python
# line 92: _connection_status 是 QLabel
self._connection_status = QLabel(locale.tr("connection_disconnected", "Disconnected"))

# line 209: 试图动画化 windowOpacity

animation.setKeyValueAt(0.5, 0.4)
animation.setEndValue(1.0)
```

**根因**：`windowOpacity` 是 `QWidget` 的**顶层窗口**属性（仅在窗口可见时有效）。`QLabel` 作为子控件没有独立的 `windowOpacity`——该属性在子控件上始终为 1.0 且不可设置。Qt 的 `QPropertyAnimation` 会静默忽略无效属性名，不抛异常。

**影响面**：`_pulse_connection_status` 在连接成功后调用（line 195），期望实现"闪烁"反馈。但动画完全无效——标签文字瞬间切换为 "Connected"，无任何视觉过渡。用户感知为生硬的状态跳变。

**修改方案**（方案 A，最小改动）：
```python
def _pulse_connection_status(self) -> None:
    effect = QGraphicsOpacityEffect(self._connection_status)
    self._connection_status.setGraphicsEffect(effect)
    animation = QPropertyAnimation(effect, b"opacity", self._connection_status)
    animation.setDuration(400)
    animation.setStartValue(1.0)
    animation.setKeyValueAt(0.5, 0.4)
    animation.setEndValue(1.0)
    animation.setLoopCount(2)
    animation.start(QPropertyAnimation.DeletionPolicy.DeleteWhenStopped)
```

```python
        "action": self.action,
        "next": self.next,
```

### [RESPONSIVE01 Low] `responsive.py:53-55` `ui_mode_for_size` 与 `is_narrow_size` 语义不一致

**位置**：`src/gui/pyqt6/responsive.py:50-59`

```python
def ui_mode_for_size(size: QSize) -> str:
    width = size.width()
    height = size.height()
    if width < BREAKPOINTS.compact_width or height < BREAKPOINTS.normal_height:
        return "compact"
    return "normal"

def is_narrow_size(size: QSize) -> bool:
```

**根因**：两个函数使用**完全相同的条件**（`width < 960 or height < 720`），但名称不同：`ui_mode_for_size` 返回 `"compact"/"normal"`，`is_narrow_size` 返回布尔值。两者语义等价但名称暗示不同判断逻辑——`is_narrow` 可能预期 `width < 800`（narrow_width）。如果未来某处调用 `is_narrow_size` 期望"极窄"判断（`width < 800`），会得到错误结果。

**影响面**：低——当前代码中 `is_narrow_size` 未被使用（grep 验证），属于 API 设计隐患。
