### TMPL-CATALOG-SILENT — `TemplateBackend.__init__` catalog 加载 bare except: pass，元素目录静默失效

**等级**: 代码质量 / 低
**位置**: `src/core/capability/element_recognition/backends/template_backend.py:67-68`

**问题代码**:

```python
# template_backend.py:62-68
self._catalog: Dict[str, Any] = {}
if catalog_path and Path(catalog_path).exists():
    try:
        with open(catalog_path, "r", encoding="utf-8") as f:
            data = json.load(f)
        self._catalog = data.get("elements", {})
    except Exception:
        pass  # ← 静默吞掉，无日志
```

**根因分析**:

`TemplateBackend.__init__` 可选加载元素目录（catalog）文件，用于提供元素定义元数据。如果 `catalog_path` 指定的 JSON 文件存在但损坏（编码错误、JSON 格式错误、磁盘截断），`json.load(f)` 抛出异常后 `except Exception: pass` 直接吞掉：

1. `self._catalog` 保持空字典 `{}`
2. 调用方无法区分"catalog 文件不存在"和"catalog 文件损坏"
3. 元素识别缺少元数据，可能影响后续的分类/过滤逻辑

**与批次 97 BACKEND-SILENT 的差异**:
- BACKEND-SILENT（批次 97）：非主识别路由匹配引擎崩溃
- TMPL-CATALOG-SILENT：初始化时 catalog 文件损坏，元素目录静默失效

**影响面**:
- **低**：catalog 是可选增强数据，核心模板匹配功能不依赖它。但元素分类和过滤可能受影响。

**修复建议**:

```python
# template_backend.py:67-68
except Exception as e:
    self._logger.warning("元素目录加载失败: %s (%s)", catalog_path, e)
```

最小修改：将 `except Exception: pass` 改为 `except Exception as e: self._logger.warning(...)`，保留空目录 fallback 但提供诊断信息。

### CONFIG-SILENT — `MaaEndControlPage._get_option_locale_data` config 读取 bare except: pass

**等级**: 代码质量 / 低
**位置**: `src/gui/pyqt6/pages/maaend_control_page.py:422-424`

**问题代码**:

```python
# maaend_control_page.py:416-424
def _get_option_locale_data(self) -> Dict[str, Any]:
    config_path = get_project_root() / "config" / "client_config.json"
    try:
        if config_path.is_file():
            data = json.loads(config_path.read_text(encoding="utf-8"))
            serial = (((data.get("device") or {}).get("last_connected")) or ((data.get("device") or {}).get("serial")))
            if serial:
                return {"serial": serial}
    except Exception:
        pass  # ← 静默吞掉，无日志
    return {}
```

**根因分析**:

`_get_option_locale_data()` 尝试从 `client_config.json` 读取设备序列号。如果文件存在但 JSON 损坏，异常被吞掉后返回空字典 `{}`：

1. 调用方收到 `{}`，无法区分"配置中无设备"与"配置文件损坏"
2. 设备自动连接逻辑可能跳过重试
3. 用户困惑：为什么设备无法自动连接

**调用链**:
```
启动 → _get_option_locale_data()
  → client_config.json 存在但损坏
  → except Exception: pass
  → return {}
  → 调用方认为无设备记录，不尝试自动连接
```

**影响面**:
- **低**：配置文件损坏是小概率事件。但一旦发生，自动连接功能静默失效。

**修复建议**:

```python
# maaend_control_page.py:422-424
except Exception as exc:
    self._logger.warning("读取 client_config.json 失败: %s", exc)
return {}
```

最小修改：将 `except Exception: pass` 改为 `except Exception as exc: self._logger.warning(...)`。

### RECOG-SILENT — `EndfieldElementRecognizer._extract_features` 特征提取 bare except: pass

**等级**: 代码质量 / 低
**位置**: `src/core/capability/element_recognition/recognizer.py:431-432`

**问题代码**:

```python
# recognizer.py:407-432
def _extract_features(self, screen: np.ndarray) -> Dict[str, Any]:
    """Extract screen features for downstream classification."""
    try:
        h, w = screen.shape[:2]
        gray = cv2.cvtColor(screen, cv2.COLOR_BGR2GRAY)
        ...
        return {
            "left_bar_brightness": round(left_bar_brightness, 1),
            "green_pixels_top_right": green_pixels_top_right,
            "full_brightness": round(float(np.mean(gray)), 1),
            "resolution": [w, h],
        }
    except Exception:
        return {}  # ← 静默返回空字典，无日志
```

**根因分析**:

`_extract_features()` 提取屏幕特征（亮度、绿色像素数等）供下游分类使用。如果 `cv2.cvtColor`、`np.mean` 等操作抛出异常（如空数组、内存不足），返回 `{}`：

1. 调用方收到空特征字典，可能误判为"无特征"而非"特征提取失败"
2. 无法区分"屏幕内容无特征"与"OpenCV 处理崩溃"
3. 下游分类逻辑可能基于不完整数据做出错误判断

**与同类代码的对比**:

| 方法 | 异常处理 | 位置 |
|------|---------|------|
| `_extract_features()` | `except Exception: return {}` | `recognizer.py:431-432` |
| `_get_color_signatures()` | 无 try/except，异常向上传播 | `recognizer.py:434+` |
| `TemplateBackend._match_single` Route 1/2 | `except Exception: pass` | `template_backend.py:248,283` |

`_get_color_signatures()`（紧邻方法）不捕获异常，允许向上传播。但 `_extract_features()` 吞掉异常并返回空字典，行为不一致。

**影响面**:
- **低**：特征是辅助数据，核心识别不依赖它。但空特征字典可能导致下游分类做出非最优决策。

**修复建议**:

```python
# recognizer.py:431-432
except Exception as exc:
    self._logger.warning("特征提取失败: %s", exc)
    return {}
```

最小修改：添加 `self._logger.warning(...)` 保留诊断信息。
