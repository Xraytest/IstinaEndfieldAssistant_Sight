## 新增发现（1 项）

### TASK-LOADER-EARLY-RETURN — `TaskLoader.load_task()` 候选路径异常时提前 `return None`，后续路径未尝试

**等级**: 代码错误 / 中
**位置**: `src/core/capability/element_recognition/tasks/task_loader.py:25-41`

**问题代码**:

```python
# task_loader.py:25-41
def load_task(self, task_name: str) -> Optional[Dict[str, Any]]:
    candidates = [
        self._tasks_root / f"{task_name}.json",          # 路径 I: assets/tasks/{name}.json
        self._tasks_root / task_name / f"{task_name}.json",  # 路径 II: assets/tasks/{name}/{name}.json
        self._maaend_tasks_root / f"{task_name}.json",   # 路径 III: 3rd-part/maaend/tasks/{name}.json
    ]
    for path in candidates:
        if path.is_file():
            try:
                with open(path, "r", encoding="utf-8") as f:
                    data = json.load(f)
                return data
            except Exception as e:
                logger.error(f"Failed to load task '{task_name}' from {path}: {e}")
                return None  # ← 提前终止，不尝试后续路径
    logger.debug(f"Task not found: {task_name}")
    return None
```

**根因分析**:

`load_task()` 设计了三层候选路径，意图是按优先级尝试多个位置加载任务定义。但 `except` 块中的 `return None` 导致异常后循环立即终止：

1. 路径 I（`assets/tasks/{name}.json`）存在但 JSON 损坏/编码错误
2. `json.load(f)` 抛出 `JSONDecodeError` 或 `UnicodeDecodeError`
3. `except Exception as e` 捕获异常，记录 error 日志
4. **`return None`** — 直接返回，路径 II 和 III **永远不尝试**
5. 调用方收到 `{"status": "error", "message": "Task '{name}' not found"}` — 消息误导（任务文件存在但损坏，不是"未找到"）

**与同类方法的对比**:

| 方法 | 异常处理策略 | 位置 |
|------|------------|------|
| `TaskLoader.load_task()` | **`return None` — 终止循环** | `task_loader.py:39` |
| `TaskLoader.load_all_tasks()` | `logger.debug(...)` + **`continue` — 继续后续** | `task_loader.py:74-75` |
| `TaskLoader.load_presets()` | `logger.debug(...)` + **`continue` — 继续后续** | `task_loader.py:99-100` |

`load_all_tasks()` 和 `load_presets()` 在同一文件中的异常处理策略是 `continue`——跳过损坏文件，继续处理其他文件。但 `load_task()` 采用了完全不同的策略：`return None`——终止搜索。

**调用链**:
```
GUI 用户选择任务 → TaskRunner.execute_task()
  → TaskLoader.tasks() → load_all_tasks() 已加载任务字典
  → task = self._task_loader.tasks().get(task_name)
  → 如果任务不在字典中（load_all_tasks 跳过了损坏文件）
  → 调用方可能需要手动调用 load_task() 重试
  → load_task() 找到路径 I 但 JSON 损坏
  → return None（路径 II/III 未尝试）
  → {"status": "error", "message": "Task 'xxx' not found"}
```

**影响面**:
- **中**：如果用户手动编辑了任务 JSON 文件导致格式错误，或磁盘写入导致文件截断，`load_task()` 会在第一个候选路径失败后放弃后续所有路径。虽然 `load_all_tasks()` 会跳过损坏文件并尝试其他目录，但单任务加载的场景（如手动重试、CLI 直接调用）会受影响。

**修复建议**:

```python
# task_loader.py:25-41
def load_task(self, task_name: str) -> Optional[Dict[str, Any]]:
    candidates = [
        self._tasks_root / f"{task_name}.json",
        self._tasks_root / task_name / f"{task_name}.json",
        self._maaend_tasks_root / f"{task_name}.json",
    ]
    for path in candidates:
        if path.is_file():
            try:
                with open(path, "r", encoding="utf-8") as f:
                    data = json.load(f)
                return data
            except Exception as e:
                logger.error(f"Failed to load task '{task_name}' from {path}: {e}")
                continue  # ← 跳过损坏路径，尝试后续候选
    logger.debug(f"Task not found: {task_name}")
    return None
```

最小修改：将 `return None` 改为 `continue`，与 `load_all_tasks()` 和 `load_presets()` 保持一致。循环结束后统一 `return None`。
