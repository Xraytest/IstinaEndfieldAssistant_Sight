## 4. `src/gui/pyqt6/theme/theme_manager.py` — 5 New Findings

### N-6 (P2) `_STYLESHEET` module variable is unused

**Line 190**: `_STYLESHEET: Optional[str] = None` is declared but never read or written. The `get_stylesheet()` function calls `_build_stylesheet()` every time and returns the freshly built string. There is no caching mechanism.

**Impact**: For a ~200-line QSS stylesheet rebuilt on every `apply_theme()` call, this creates unnecessary garbage and CPU load.

**Recommendation**: Either remove `_STYLESHEET` and accept the rebuild cost, or implement actual caching:
```python
_STYLESHEET: Optional[str] = None
_CURRENT_THEME: Optional[str] = None

def get_stylesheet(theme_name: str = "arknight") -> str:
    global _STYLESHEET, _CURRENT_THEME
    if _STYLESHEET is not None and _CURRENT_THEME == theme_name:
        return _STYLESHEET
    _STYLESHEET = _build_stylesheet(THEMES.get(theme_name, THEMES["arknight"])["colors"])
    _CURRENT_THEME = theme_name
    return _STYLESHEET
```

### N-7 (P1) Thread-unsafe singleton with dual creation paths

**Lines 393-402**:
```python
class ThemeManager:
    _instance: Optional["ThemeManager"] = None

    def __new__(cls) -> "ThemeManager":
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

    @classmethod
    def get_instance(cls) -> "ThemeManager":
        if cls._instance is None:
            cls._instance = cls()
        return cls._instance
```

Both `__new__` and `get_instance` check `cls._instance is None` without locking. If two threads call `ThemeManager()` or `get_instance()` simultaneously, both may create separate instances. `get_instance()` can overwrite `__new__`'s instance, or vice versa.

**Impact**: In the GUI's parallel startup design, theme access could race. Two `ThemeManager` instances with independent `_current_theme` state could cause theme flickering or inconsistent colors.

**Recommendation**: Make singleton creation thread-safe:
```python
    def __new__(cls) -> "ThemeManager":
        if cls._instance is None:
            with threading.Lock():
                if cls._instance is None:
                    cls._instance = super().__new__(cls)
        return cls._instance
```

### N-8 (P1) Global mutable state (`COLORS`, `_current_theme`) accessed without locking

**Lines 445-453**:
```python
    def set_current_theme(self, theme_name: str) -> None:
        global COLORS
        if theme_name in THEMES:
            COLORS.clear()
            COLORS.update(THEMES[theme_name]["colors"])
            global _current_theme
            _current_theme = theme_name
```

And **lines 404-405**:
```python
    @property
    def colors(self) -> Dict[str, str]: return COLORS
```

Any thread reading `COLORS` while `set_current_theme` is mutating it sees a torn read (partially cleared dict, or mixed colors from two themes).

**Recommendation**: Either make `COLORS` immutable per-theme and atomically swap a reference, or add a lock around all accesses.

### N-9 (P1) `ensure_app_fonts` mutates global `FONTS` without locking

**Lines 467-497**: `ensure_app_fonts()` reads and writes `FONTS["family"]` and `FONTS["family_display"]` without any synchronization. If called concurrently (e.g., during parallel GUI startup), two threads could both see `_FONT_RESOURCES_LOADED = False` and both enter the font-loading logic.

**Recommendation**: Add a module-level lock around the font loading block, or move font loading to the single-threaded startup path.

### N-10 (P2) `addApplicationFont` may add duplicate fonts

**Line 484**: `font_id = QFontDatabase.addApplicationFont(str(path))` is called for each candidate path. If the same font file is already registered, Qt may return a new `font_id` each time, or it may deduplicate silently. The code doesn't check whether the font is already loaded before calling `addApplicationFont`.

**Impact**: Minor — wasted work and potential memory bloat if Qt doesn't deduplicate.

**Recommendation**: Cache the result per font family, or check `QFontDatabase.families()` first.

## 10. `src/gui/pyqt6/scripting/player.py` — 5 New Findings

### N-23 (P2) `play()` does not stop existing timer

**Lines 52-64**: `play()` sets `_current_index = 0` and calls `_schedule_next()`, but does **not** stop any existing `_action_timer`. If `play()` is called while a previous playback is running, two timers fire concurrently, interleaving actions.

**Recommendation**: Stop any existing timer at the start of `play()`:
```python
    def play(self, delay_ms: Optional[int] = None) -> None:
        self.stop()  # Stop any existing playback
        ...
```

### N-24 (P2) `is_playing()` returns incorrect state during timer wait

**Line 87-88**:
```python
    def is_playing(self) -> bool:
        return self._script is not None and not self._stopped and self._current_index < len(self._script.actions)
```

`_current_index` is incremented in `_execute_action` **before** the action runs. After incrementing to `len(self._script.actions)`, `is_playing()` returns `False` even though the timer is still active and will call `_on_finished` on the next tick.

**Impact**: UI code that checks `is_playing()` (e.g., `scripting_page.py` line 185) may think playback has finished when the timer is still pending.

**Recommendation**: Track `_timer_active` separately, or check `self._action_timer is not None and self._action_timer.isActive()`.

### N-25 (P2) `_find_widget` uses ambiguous `object_name` matching

**Lines 133-143**:
```python
        for widget in app.topLevelWidgets():
            found = widget.findChild(QWidget, action.object_name)
            if found is not None:
                return found
```

If multiple widgets share the same `object_name`, `findChild` returns the first match in `QObject` tree order. This may not be the widget the user interacted with during recording.

**Recommendation**: Consider recording the widget's class name and a path of parent `object_name`s, then matching on the full path.

### N-26 (P2) `_do_click` coordinate roundtrip is a no-op

**Lines 145-154**:
```python
        pos = QPoint(int(x), int(y))
        local_pos = widget.mapFromGlobal(widget.mapToGlobal(pos))
        QTest.mouseClick(widget, Qt.MouseButton.LeftButton, pos=local_pos)
```

The recorded `x, y` are local coordinates (from `event.position()`). The `mapToGlobal` → `mapFromGlobal` roundtrip is mathematically a no-op when the widget geometry hasn't changed. However, if the widget has **moved or resized** between recording and playback (e.g., window was dragged, DPI changed), the click lands at the old local coordinates, which may be outside the widget or in the wrong spot.

**Impact**: Replay is fragile across window state changes or multi-monitor DPI differences.

**Recommendation**: Store the global position at record time, then replay using `QTest.mouseClick` at that global position (or map to the widget's current geometry).

### N-27 (P3) `stop()` does not clear `_action_timer` reference

**Lines 77-85**: `stop()` sets `_stopped = True` and calls `self._action_timer.stop()`, but does **not** set `self._action_timer = None`. The stale `QTimer` reference remains.

**Impact**: Minor — Qt handles the timer cleanup, but `is_playing()` or other checks that inspect `_action_timer` may behave unexpectedly.

**Recommendation**: Set `self._action_timer = None` after stopping.

## 11. `src/gui/pyqt6/scripting/recorder.py` — 2 New Findings

### N-28 (P2) `_install_app_filter` can leave stale `_app_filter_installed` flag

**Lines 111-116**:
```python
    def _install_app_filter(self) -> None:
        app = QApplication.instance()
        if app is not None and not self._app_filter_installed:
            app.installEventFilter(self)
            self._app_filter_installed = True
```

If `QApplication.instance()` returns `None`, `_app_filter_installed` remains `False`. Good.

**Lines 118-126**:
```python
    def _uninstall_app_filter(self) -> None:
        app = QApplication.instance()
        if app is not None and self._app_filter_installed:
            try:
                app.removeEventFilter(self)
            except Exception:
                pass
            self._app_filter_installed = False
```

If `app` becomes `None` **after** the filter was installed (e.g., during app shutdown), the condition `app is not None` fails, the filter is **not removed**, and `_app_filter_installed` remains `True`. **Impact**: Event filter leaks into a new QApplication instance, or the recorder cannot be restarted after shutdown.

**Recommendation**: Decouple the flag from `app` availability, or reset `_app_filter_installed = False` unconditionally in `stop()`.

### N-29 (P3) `_should_skip` returns `True` for widgets without `objectName`

**Lines 128-139**: Widgets without an `objectName` are skipped. This is intentional for internal Qt widgets, but it means user-created widgets that lack an `objectName` are silently ignored during recording, with no warning.

**Recommendation**: Log a debug message when a widget is skipped due to missing `objectName`, to help users configure their UI for recording.
