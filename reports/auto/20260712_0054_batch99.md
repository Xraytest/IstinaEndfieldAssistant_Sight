### SCRIPT-SAVE-NONATOMIC — `Script.save()` 非原子写入，中断导致 JSON 文件损坏

**等级**: 代码质量 / 低
**位置**: `src/gui/pyqt6/scripting/models.py:49-54`

**问题代码**:

```python
# models.py:49-54
def save(self, directory: Path) -> Path:
    """Save a script to a JSON file in the given directory."""
    directory.mkdir(parents=True, exist_ok=True)
    path = directory / f"{self.name}.json"
    path.write_text(json.dumps(self.to_dict(), ensure_ascii=False, indent=2), encoding="utf-8")
    return path
```

**根因分析**:

`Script.save()` 直接调用 `path.write_text()` 将 JSON 写入目标路径。如果写入过程中程序崩溃（断电、强制关闭、异常），目标文件会停留在**部分写入状态**——包含截断或不完整的 JSON。

**同类代码对比**:

| 方法 | 写入方式 | 位置 |
|------|---------|------|
| `Script.save()` | **直接写入目标路径** | `models.py:53` |
| `QueueState.persist()` | **tmp + os.replace 原子写入** | `queue_state.py:122-124` |
| `runtime.save_config()` | 非原子（DUP-A 已记录） | `runtime.py` |
| `maaend_control_page._export_queue()` | 非原子（DUP-A 已记录） | `maaend_control_page.py` |

同一代码库中，`QueueState.persist()` 已使用正确的原子写入模式：

```python
# queue_state.py:122-124 — 正确范例
tmp_path = self._state_path.with_suffix(".tmp")
tmp_path.write_text(json.dumps(state, ...), encoding="utf-8")
os.replace(tmp_path, self._state_path)
```

但 `Script.save()` 未遵循同一模式。这表明原子写入知识未在项目内传播——开发者在一处修好了问题，但在另一处又犯了同样的错误。

**影响面**:
- **低**：脚本文件体积小（通常 <10KB），崩溃窗口极短。但损坏的 JSON 会在下次加载时导致 `json.loads()` 崩溃（`Script.load()` 无 try/except）。

**修复建议**:

```python
# models.py:49-54
def save(self, directory: Path) -> Path:
    """Save a script to a JSON file in the given directory."""
    directory.mkdir(parents=True, exist_ok=True)
    path = directory / f"{self.name}.json"
    tmp_path = path.with_suffix(".tmp")
    tmp_path.write_text(json.dumps(self.to_dict(), ensure_ascii=False, indent=2), encoding="utf-8")
    os.replace(tmp_path, path)
    return path
```

与 `QueueState.persist()` 使用完全相同的 tmp+replace 模式。`os.replace()` 在 Windows 上是原子操作。

### SCRIPT-LIST-SILENT — `_refresh_script_list()` bare except: pass 静默吞掉加载错误

**等级**: 代码质量 / 低
**位置**: `src/gui/pyqt6/scripting/scripting_page.py:139-140`

**问题代码**:

```python
# scripting_page.py:139-140
for path in files:
    try:
        script = Script.load(path)
        item = QListWidgetItem(f"{script.name} ({len(script.actions)} actions)")
        item.setData(Qt.ItemDataRole.UserRole, str(path))
        self._script_list.addItem(item)
    except Exception:
        pass  # ← 静默吞掉，无日志
```

**根因分析**:

`_refresh_script_list()` 遍历 `scripts/recorded/*.json` 文件列表，逐个调用 `Script.load()` 加载。如果某个 JSON 文件损坏（被手动编辑、磁盘写入中断、编码错误），`Script.load()` 会抛出 `json.JSONDecodeError` 或其他异常。但 `except Exception: pass` 直接吞掉，调用方看不到任何反馈：

1. 损坏的脚本文件从列表中消失
2. 用户困惑："我明明保存了这个脚本，为什么列表中没有？"
3. 无日志记录，开发者无法定位

**同类代码对比**:

| 文件 | 位置 | 异常处理 |
|------|------|---------|
| `ScriptingPage._refresh_script_list` | `scripting_page.py:139-140` | `except Exception: pass` |
| `TemplateBackend._match_single Route 1` | `template_backend.py:248-249` | `except Exception: pass` |
| `TemplateBackend._match_single Route 2` | `template_backend.py:283-284` | `except Exception: pass` |
| `ColorBackend._match_one_signature Route 1` | `color_backend.py:244-245` | `except Exception: pass` |

BACKEND-SILENT 模式（批次 97）的又一个新实例，但位于不同的模块和上下文。

**与 BACKEND-SILENT 的差异**:
- BACKEND-SILENT（批次 97）：识别后端，静默吞错导致无法区分"模板不存在"与"引擎崩溃"
- SCRIPT-LIST-SILENT：脚本列表加载，静默吞错导致损坏文件消失

**调用链**:
```
用户打开"脚本"页面
  → ScriptingPage._refresh_script_list()
    → 遍历 scripts/recorded/*.json
    → Script.load(corrupted_file.json) → json.JSONDecodeError
    → except Exception: pass
    → 损坏文件从列表中消失，无提示
```

**影响面**:
- **低**：仅影响脚本列表显示，不影响核心自动化功能。损坏的脚本文件可以被用户删除后恢复。

**修复建议**:

```python
# scripting_page.py:139-141
for path in files:
    try:
        script = Script.load(path)
    except Exception as exc:
        self._logger.warning("脚本加载失败，已跳过: %s (%s)", path.name, exc)
        continue
    item = QListWidgetItem(f"{script.name} ({len(script.actions)} actions)")
    item.setData(Qt.ItemDataRole.UserRole, str(path))
    self._script_list.addItem(item)
```

最小修改：将 `except Exception: pass` 改为 `except Exception as exc: self._logger.warning(...); continue`，保留"跳过损坏文件"的行为但提供诊断信息。
