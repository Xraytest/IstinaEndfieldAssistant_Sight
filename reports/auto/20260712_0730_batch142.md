## 新增发现

### UX-01（NEW）— screen_off_timeout 状态泄漏

**严重度**: Medium（影响用户体验，非崩溃/安全漏洞）

**位置**: `android_runtime.py:210`

**问题描述**: `_start_server()` 中新增了 `settings put system screen_off_timeout 1800000`（30 分钟）写入，但 `_cleanup()`（L398-421）中**未恢复原始值**。scrcpy 会话结束后，用户原有的屏幕超时设置被永久覆盖为 30 分钟。

**代码证据**:
```python
# L207-212 — 写入（新增）
try:
    self._host_shell("settings put system screen_off_timeout 1800000")
except Exception:
    pass

# L398-421 — 清理（现有）
def _cleanup(self) -> None:
    self._close_codec()
    self._last_frame_ts = 0.0
    # ... 仅清理 port forwarding、server process、codec
    # 未恢复 screen_off_timeout
```

**影响**: 用户原有屏幕超时设置（可能为 30s、1min、5min 等）被静默修改为 30 分钟，且不会自动恢复。若用户未察觉，后续使用体验持续受影响。

**修改建议**: 在写入前通过 `settings get system screen_off_timeout` 保存原始值，在 `_cleanup()` 中恢复：
```python
# _start_server() 写入前
try:
    self._original_screen_timeout = self._host_shell("settings get system screen_off_timeout").strip()
    self._host_shell("settings put system screen_off_timeout 1800000")
except Exception:
    pass

# _cleanup() 恢复
if hasattr(self, "_original_screen_timeout") and self._original_screen_timeout:
    try:
        self._host_shell(f"settings put system screen_off_timeout {self._original_screen_timeout}")
    except Exception:
        pass
```
